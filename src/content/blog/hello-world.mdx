I've been putting off building a personal site for years. Every time I started one, it felt like a vanity project that would eat a weekend and never get finished.

So what changed?

## The honest reason

I'm targeting roles at AI companies — startups and the larger players — and I wanted somewhere to point that wasn't just a LinkedIn profile. Not to check a box, but because I genuinely have things I want to figure out in public.

The LLM ecosystem is moving fast enough that half of what I learn in a month is obsolete the next. Writing about it — even rough notes — forces me to slow down and actually understand what I'm building with, not just ship something that works.

## What this site is

This is a portfolio and a blog, built with [Astro](https://astro.build). The goal is:

- **Show depth** — I want the code itself to be something I'd be proud of, not a Bootstrap template.
- **Document the journey** — Posts here will be engineering notes: things I figured out, things that surprised me, things I got wrong.
- **Stay low-maintenance** — Static site, MDX posts, no CMS. I want to write in my editor.

## What I'll write about

Mostly:

- Building with LLMs (RAG, evals, prompt engineering, tool use)
- TypeScript and Python patterns I find useful
- Architecture decisions and the reasoning behind them
- Occasional deep dives into open source tools I'm using

I'm a mid-level engineer — not an expert, not a beginner. These posts will reflect that honestly.

## The stack

In case you're curious: Astro 6, React islands for anything interactive, MDX for this blog, CSS custom properties for styling (no Tailwind). The full reasoning is in the repo if you want to read it.
